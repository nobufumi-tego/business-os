# 共通の振る舞いルール（core/rules.md）

すべてのスラッシュコマンド・ユースケースが共有する基本ルール。

## 言語
- ユーザーへの応答は必ず日本語
- ファイル名は日本語OK

## 対話の作法
- 専門用語を使わない。必要なら例を添える
- 1問1答で進める（同時に複数質問しない）
- 答えに迷っているようなら具体例を3つ提示
- ユーザーが疲れていそうなら「ここで休みますか？」と聞く

## 出力の規約
- 成果物は必ず `outputs/<カテゴリ>/YYYYMMDD_<件名>.yaml` または `.md`
- 対話の中で消えないように、その場でファイル保存する
- 保存ファイルパスを毎回ユーザーに伝える

## Git の規約
- ローカルのみ運用。`git push` `git remote add` は提案しない
- セクション完了時に `/保存` を案内
- inputs/ 配下は絶対に `git add` しない（permissions で deny 済）

## 機密データ
- inputs/ 配下のファイルを AI に送る前に必ず確認を取る
- 個人名・取引先名が含まれる場合は、伏せ字化を提案

## エラー時
- スタックトレースをそのまま見せない
- 日本語で「何が起きたか」「どうすれば直るか」を簡潔に説明
- 解決できなければ docs/困ったとき.md を案内

## 成果先行・解説後回し（UX原則）
- 仕組みの説明から入らない
- まず3問で骨格を作って成果ファイルを出す
- 種明かしは docs/しくみ.md に分離
