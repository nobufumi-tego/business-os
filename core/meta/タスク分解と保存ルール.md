# タスク分解と保存ルール（全ユースケース共通）

## タスク分解
- 1ユースケース = 5〜10サブタスク（多すぎても少なすぎてもダメ）
- 各サブタスクは「1セクション分の質問群」が目安
- TaskCreate / TaskUpdate でユーザーに進捗を可視化する

## 自動コミット（保存）
- セクション完了時に「ここまでを保存しますか？」とユーザーに確認
- YES なら：
  - `git add <変更されたファイル>`（inputs/ は除外）
  - コミットメッセージは Claude が生成
  - 形式: `<ユースケース>: <対象> - <セクション名>を記録`
- NO なら次に進む（ユーザーが `/保存` を打つまで未保存）

## コミットメッセージの形式

`<ユースケース名>: <対象> - <セクション名>を記録`

- `<ユースケース名>` は `usecases/` 配下のフォルダ名と揃える
- `<対象>` はユーザーが棚卸ししている対象（業務名・フォルダ名・システム名 等）
- `<セクション名>` は今完了したセクション（例: 基本情報 / 工程 / 重複ファイル候補 等）

具体例は各 usecase の `definition.md` 側で示す。core/ では特定のユースケース名に依存しない。

## 粒度の切り替え
ユーザーが以下を打ったら粒度を切り替える：
- `/細かく` … サブタスクごとにコミット（履歴詳細・巻き戻しやすい）
- `/標準` … セクション完了ごと（推奨）
- `/ざっくり` … ユースケース完了時に1回だけ

## セッション終了時の保護
- Claude セッションが終わるとき、未保存の変更があれば自動コミット
- メッセージ: "自動保存: セッション終了時の未保存変更"
- これは Stop hook で実装（settings.json）
