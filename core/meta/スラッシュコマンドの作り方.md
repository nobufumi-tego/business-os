# スラッシュコマンドの作り方

## ファイル配置
- `.claude/commands/<日本語名>.md` に置く
- ファイル名がそのままスラッシュコマンドになる
  - 例: `業務棚卸し.md` → `/業務棚卸し`

## ファイル構造

```markdown
---
description: コマンドの一行説明（日本語）
---

@core/rules.md
@core/meta/タスク分解と保存ルール.md
@usecases/<対象>/definition.md

このコマンドが起動したときに行う指示を、
Claude への手紙のように日本語で書く。
```

## 設計のコツ

- **薄く保つ**: コマンドファイル本体は短く。実体は `usecases/<名前>/definition.md` に書く
- **`@` インポートで繋ぐ**: rules.md・definition.md を必ずインポート
- **最初の出力を早く**: ユーザーが3問答えれば最初のファイルが出来るように設計

## カスタムコマンドを作る手順

1. `usecases/<名前>/definition.md` を作る
2. `usecases/<名前>/template.yaml` を作る（必要なら）
3. `.claude/commands/<名前>.md` を作る
4. `claude` を再起動して `/<名前>` で動作確認
